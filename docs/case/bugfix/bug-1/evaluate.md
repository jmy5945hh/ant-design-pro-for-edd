# BUG-1 修复评分标准

**编号**：BUG-1 | **难度**：⭐⭐ | **满分**：10 分

---

## 评分维度

| 维度 | 权重 | 说明 |
|------|------|------|
| 定位准确度 | 30% | 是否准确找到问题代码行 |
| 修复正确性 | 40% | 修复后数据展示是否正确 |
| 根因理解 | 20% | 能否解释为什么会出现这个 Bug |
| 预防意识 | 10% | 是否检查了关联代码防止同类问题 |

---

## 评分细则

### 0-2 分：未发现或修复无效

| 分数 | 表现 |
|------|------|
| 0 | 未提交修复，或修复内容与本题无关 |
| 1 | 定位到了文章列表页面，但修改了无关的代码（如改了样式、改了其他组件） |
| 2 | 找到了 `<IconText>` 组件区域，但修改了错误的字段（如把 `item.like` 改成了 `item.message`） |

### 3-5 分：找到了问题但修复不完整

| 分数 | 表现 |
|------|------|
| 3 | 发现收藏数显示的是评论数的值，但把 `item.star` 改成了硬编码数字（如 `text={100}`），没有真正修复字段映射 |
| 4 | 正确将 `text={item.message}` 改回 `text={item.star}`，但无法解释为什么要这样改 |
| 5 | 修复正确，能说明「收藏的 text 绑错了数据字段」，但解释不够深入（未提及数据模型） |

### 6-8 分：正确修复 + 理解根因

| 分数 | 表现 |
|------|------|
| 6 | 修复正确 + 能解释 `ListItemDataType` 中 `star`、`like`、`message` 是三个独立字段，以及为什么 TypeScript 没报错（三个都是 `number`） |
| 7 | 在 6 分基础上，主动检查了同组另外两个 `<IconText>`（like、message）的字段绑定确认它们是正确的 |
| 8 | 在 7 分基础上，能总结这类 Bug 的产生原因（copy-paste 后忘记改字段名），提出 Code Review 中应关注同类问题的建议 |

### 9-10 分：卓越修复

| 分数 | 表现 |
|------|------|
| 9 | 修复正确 + 主动验证了三个字段都正确 + 提出改进建议（如封装一个类型安全的 `<ArticleStatItem>` 组件，用 TypeScript 约束 `type` 与数据字段的对应关系） |
| 10 | 在 9 分基础上，编写了可验证修复的测试用例（或以其他方式证明修复无副作用），并检查了项目中其他使用 `<IconText>` 的地方是否存在同类问题 |

---

## 验收检查项

修复完成后，评委将验证以下内容：

- [ ] 页面加载后，星形图标显示的数字与消息图标显示的数字**不同**
- [ ] 三个图标显示的数字分别与 API 返回的 `star`、`like`、`message` 字段对应
- [ ] 修复未引入新的 TypeScript 编译错误或 Lint 告警
- [ ] 其他页面的 `<IconText>` 使用未受影响
