# BUG-1：文章列表收藏数显示错误

**编号**：BUG-1

**页面路径**：列表 → 搜索列表 → 文章（`/list/search/articles`）

**发现版本**：竞赛分支

**严重程度**：一般（数据展示错误，不影响功能）

**复现率**：100%

---

## 测试环境

- 浏览器：Chrome 最新版
- 设备：桌面端（1920×1080）

---

## 测试步骤

1. 启动项目，进入「列表」→「搜索列表」→「文章」页面
2. 页面加载完成后，观察文章列表中每篇文章底部的操作栏
3. 操作栏从左到右有三个图标：
   - 星形图标（⭐，收藏数）
   - 心形图标（👍，点赞数）
   - 消息图标（💬，评论数）
4. 分别记录每篇文章三个图标旁边的数字

---

## 预期结果

三个图标分别显示各自独立的统计数值：
- 星形图标 ⭐ 旁边显示**收藏数**（star）
- 心形图标 👍 旁边显示**点赞数**（like）
- 消息图标 💬 旁边显示**评论数**（message）

三个数字应当互不相同（或至少不完全相同）。

---

## 实际结果

**星形图标（收藏）显示的数字与消息图标（评论）的数字完全相同。**

例如某篇文章：
- 星形 ⭐：`28`（和评论数一样）
- 心形 👍：`15`
- 消息 💬：`28`

收藏数和评论数明显应该是两个独立的统计指标，但页面上却展示了相同的数值。心形图标的数字倒是正常的。

---

## 附加信息

- 该问题在所有文章条目上均存在，不是个别数据异常
- 页面首次加载即可复现，无需任何交互操作
- Network 面板中 API 返回的原始数据是正常的（三个字段值不同），但页面渲染时展示错了
- 未发现控制台报错
