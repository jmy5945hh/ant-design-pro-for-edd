# BUG-4：监控页活动图表异常高频刷新

**编号**：BUG-4

**页面路径**：仪表盘 → 监控页（`/dashboard/monitor`）→ 右侧「活动情况预测」区域

**发现版本**：竞赛分支

**严重程度**：严重（性能问题，影响页面可用性）

**复现率**：100%

---

## 测试环境

- 浏览器：Chrome 最新版
- 设备：桌面端（1920×1080）

---

## 测试步骤

1. 进入「仪表盘」→「监控页」
2. 找到页面右侧的「活动情况预测」卡片区域
3. 保持页面静止，不做任何操作，**持续观察面积图的变化**约 10 秒
4. 注意观察：
   - 图表曲线的更新频率
   - 曲线形状和数据值的稳定性
   - 页面滚动的流畅度
5. （可选）打开 Chrome DevTools → Performance 面板，录制 3-5 秒，观察火焰图

---

## 预期结果

1. 图表按合理的时间间隔（如 1-2 秒）平滑更新
2. 数据值在合理范围内波动，视觉上可阅读
3. 页面滚动流畅，无卡顿
4. CPU 占用保持在正常水平

---

## 实际结果

1. ❌ **图表每秒更新约 10 次**（约每 100ms 一次），远超正常频率
2. ❌ 面积图曲线**剧烈跳动**，数据值快速闪烁变化，完全无法阅读具体数值
3. ❌ 页面滚动出现**明显卡顿**，浏览器 CPU 占用显著升高
4. ❌ 「目标评估」右侧的统计数值（`xxx 亿元`）也在快速跳动

**Performance 面板分析**：
- 录制 3 秒可观察到密集的 `setTimeout` 回调触发
- React 渲染火焰图中可看到 `ActiveChart` 组件高频重渲染
- 每次渲染都触发了 `getActiveData()` 生成新数据

---

## 附加信息

- 该图表组件位于 `src/pages/dashboard/monitor/components/ActiveChart/`
- 使用 `@ant-design/plots` 的 `Area` 组件渲染面积图
- 数据通过 `setTimeout` 递归定时更新
- 「目标评估」显示的文字是静态的，但最大/中位数在随图表数据变化
