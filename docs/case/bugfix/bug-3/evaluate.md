# BUG-3 修复评分标准

**编号**：BUG-3 | **难度**：⭐⭐ | **满分**：10 分

---

## 评分维度

| 维度 | 权重 | 说明 |
|------|------|------|
| 定位准确度 | 30% | 是否找到 `onFinish` 回调中的 `setCurrent` 调用 |
| 修复正确性 | 35% | 修复后「再转一笔」是否正确回到第一步 |
| 根因理解 | 20% | 能否解释 StepsForm 的 0-based 索引设计 |
| 完整性检查 | 15% | 是否验证了表单重置等关联逻辑 |

---

## 评分细则

### 0-2 分：未发现或修复无效

| 分数 | 表现 |
|------|------|
| 0 | 未提交修复，或修复内容与本题无关 |
| 1 | 定位到了 `StepResult` 组件但修改了无关内容（如改了按钮文案、按钮样式） |
| 2 | 意识到了步骤索引有问题，但改错了值（如改成 `setCurrent(2)` 跳到了第三步，或改成 `setCurrent(-1)` 导致崩溃） |

### 3-5 分：找到了问题但修复不完整

| 分数 | 表现 |
|------|------|
| 3 | 正确将 `setCurrent(1)` 改回 `setCurrent(0)`，但无法解释为什么是 0 而不是 1 |
| 4 | 修复正确 + 能说出「StepsForm 的 current 从 0 开始」，但未检查 `form.resetFields()` 是否也在同一回调中 |
| 5 | 修复正确 + 能解释 0-based 索引，并确认了 `form.resetFields()` 的作用——清空表单数据，确保第一步看到的是空白表单 |

### 6-8 分：正确修复 + 深入理解

| 分数 | 表现 |
|------|------|
| 6 | 在 5 分基础上，能追踪完整的状态流转：`setCurrent(0)` → StepsForm 切换到 Step 0 → `form.resetFields()` 清空 Step 0 的表单 → 用户看到空白的第一步 |
| 7 | 在 6 分基础上，能验证 StepsForm 的 `current` prop 与 `onCurrentChange` 回调的配合关系，确认修改不会影响其他步骤的切换 |
| 8 | 在 7 分基础上，能指出如果只改 `setCurrent` 而忘记调 `resetFields`，会有什么问题（第一步显示旧数据），以及为什么当前代码已经正确在同一个回调中处理了（展示了对完整性的关注） |

### 9-10 分：卓越修复

| 分数 | 表现 |
|------|------|
| 9 | 修复正确 + 提出改进建议（如用枚举常量 `STEP_FILL / STEP_CONFIRM / STEP_RESULT` 替代魔数 0/1/2，消除此类索引错误的可能性） |
| 10 | 在 9 分基础上，检查了项目中其他使用 StepsForm 或类似步骤组件的地方是否存在同类索引错误，并给出了排查结论 |

---

## 验收检查项

修复完成后，评委将验证以下内容：

- [ ] 完成一次完整的转账流程后，点击「再转一笔」，正确回到第一步「填写转账信息」
- [ ] 回到第一步后，表单字段已重置为初始值或空白状态
- [ ] 步骤条高亮第一步
- [ ] 正常的「下一步」「上一步」流程不受影响
- [ ] 修复未引入 TypeScript 编译错误或 Lint 告警
