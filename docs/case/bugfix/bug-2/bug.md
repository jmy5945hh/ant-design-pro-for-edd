# BUG-2：批量删除后表格数据不刷新

**编号**：BUG-2

**页面路径**：列表 → 查询表格（`/list/table-list`）

**发现版本**：竞赛分支

**严重程度**：严重（数据操作后界面不更新，用户以为删除失败会重复操作）

**复现率**：100%

---

## 测试环境

- 浏览器：Chrome 最新版
- 设备：桌面端（1920×1080）

---

## 测试步骤

1. 启动项目，进入「列表」→「查询表格」页面
2. 等待表格数据加载完成（约 50 条数据）
3. 勾选表格中前 2-3 条数据（点击行首复选框）
4. 点击底部出现的「批量删除」按钮
5. 观察页面变化：
   - 提示消息内容
   - 表格中刚才勾选的数据是否还存在
6. 打开 Chrome DevTools → Network 面板，重复步骤 3-5，观察网络请求

---

## 预期结果

1. 点击「批量删除」后，提示消息应显示「删除成功」
2. 表格数据应**自动刷新**，被删除的记录不再出现在列表中
3. 底部批量操作栏应消失（因为已无选中项）
4. 手动刷新页面后，被删除的数据确实不在列表中（确认后端已删除）

---

## 实际结果

1. ✅ 提示消息正确显示「Deleted successfully and will refresh soon」
2. ❌ **表格中的数据没有刷新**——被勾选删除的那几条记录仍然显示在表格中
3. ✅ 底部批量操作栏确实消失了（因为 `selectedRows` 已被清空）
4. ❌ 需要**手动刷新页面**（F5）或**切换页面**后，被删除的数据才会从表格中消失

**补充**：Network 面板显示 DELETE 请求成功返回 200，后端确实完成了删除操作。但前端表格没有主动重新请求数据。
