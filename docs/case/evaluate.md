# 前端开发能力评分表 — /chatbot 页面复现评估

> 评估对象完成 `/chatbot` AI 智能对话页面的复现（基于 Ant Design Pro v6 + Umi Max 4 + antd v6 + Ant Design X）。
>
> 评分标准：每项 **1–5 分**（1=未实现/严重错误, 3=基本达标, 5=超出预期/优雅实现），总分 **80 分**。

---

## 二、UI 布局与组件实现（20 分）

| # | 评估维度 | 评分要点 | 得分 | 备注 |
|---|---------|---------|------|------|
| 2.1 | **整体布局** | `PageContainer ghost` + Card(borderless) 双层容器；左侧 Sidebar + 右侧 Main 的二分布局；高度固定 `calc(100vh - 160px)`，无页面级滚动条 | /4 | |
| 2.2 | **会话列表（Sidebar）** | 使用 `@ant-design/x` 的 `Conversations` 组件；支持"新建对话"按钮；会话按"今天/昨天/更早"分组；当前会话高亮；草稿会话标记 (`isDraft`) | /4 | |
| 2.3 | **消息区（Bubble）** | 使用 `@ant-design/x` 的 `Bubble` 组件；用户消息右对齐（蓝色系）、AI 消息左对齐（浅色系）；消息列表自动滚动；最大宽度 940px 居中 | /4 | |
| 2.4 | **输入区（Sender）** | 使用 `@ant-design/x` 的 `Sender` 组件；多行输入 (minRows=4, maxRows=8)；Enter 发送、Shift+Enter 换行；请求中显示 loading + 取消按钮；最大宽度 940px | /4 | |
| 2.5 | **空状态引导** | 空消息时显示打字机欢迎词（逐字播放 80ms）；输入框居中底部 (`footerCenter`)；有消息后沉底 (`footer`) | /4 | |

## 三、AI 交互与数据流（16 分）

| # | 评估维度 | 评分要点 | 得分 | 备注 |
|---|---------|---------|------|------|
| 3.1 | **流式 AI 请求** | 使用 `@ant-design/x-sdk` 的 `useXChat` / `OpenAIChatProvider` / `XRequest`；POST SSE 流式请求到 `CHAT_API_URL`；`model: glm-4.5-flash, stream: true` | /4 | |
| 3.2 | **多轮对话** | 同一会话内消息累积传递（`messages` 历史）；切换会话时消息隔离（`conversationKey`）；无需手动维护消息数组 | /4 | |
| 3.3 | **Markdown 渲染** | AI 回复使用 `XMarkdown` 渲染；流式更新时使用流式渲染模式；打字机动效 (`typing: step:2, interval:20`) | /4 | |
| 3.4 | **Think 深度思考解析** | 解析 `<think>...</think>` 标签；完整格式 → think 区域折叠显示 + 主回复；流式不完整格式 → 仅显示 think 区域 | /4 | |
